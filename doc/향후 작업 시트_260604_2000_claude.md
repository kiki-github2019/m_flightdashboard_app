### 3. 알려진 잠재 위험 / 회귀 가능성

| ID | 위험 | 우선순위 | 상태 | 작업 내용 |
|---|---|---|---|---|
| R-01 | `LastDisplayedFrame` vs `LastDecodedFrame` 분리 후 일반 드래그 회귀 가능성 | **P1** | 회귀 위험 | E01 시나리오에서 frame skip / 이중 디코딩 발생 여부 확인 |
| R-02 | `restoreBoardPanelState` 가 snapshot.PanelVisible 무시 → 매우 오래된 사용자가 기대하는 "원복" 동작과 다를 수 있음 | P1 | 동작 변경 | UX 가이드 추가 또는 명시적 "원복" 버튼 |
| R-03 | `prefetchFrameToCacheOnly` 가 VideoReader 공유 → 동기 decode 와 race condition 가능성 (MATLAB single-thread 이므로 실질 안전) | P2 | 검토 | 빠른 드래그 + prefetch 동시 시나리오 |
| R-04 | `capturePlotConfigFromUi` identity 매칭 (flight, tab, order) → drag-reorder 시 YColumn 가 잘못 연결될 수 있음 | P2 | 명시된 한계 | drag-reorder UI 추가 시 매칭 키 재설계 필요 |
| R-05 | `loadAviFileFromPath(preserveSync=true)` 가 새 AVI 의 TotalFrames 이 다를 경우 anchor 가 범위 밖이 될 수 있음 | P2 | 미처리 | clamp + 사용자 경고 |

| R-07 | 50 케이스 × baseline 캡처로 ~300+ PNG → MATLAB Drive quota | P2 | 모니터 | exportapp 해상도 축소 옵션 |
